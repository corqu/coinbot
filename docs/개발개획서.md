# 코인 선물 거래 자동화 시스템 (CoinBot) 개발 계획서

## 1. 프로젝트 개요 (Overview)

### 목표

개인별 맞춤형 투자 전략을 생성하고, 이를 기반으로 실시간 시그널을 포착하여 바이비트(Bybit) 거래소에서 선물 거래를 수행하는 자동화 시스템 구축.

### 핵심 가치

사용자별 전략 커스터마이징 및 자동 매매를 통한 투자 수익 극대화 및 편의성 제공.

| **Client**        | React / Web        | _ 유저 인터페이스 제공 [cite: 2] <br> _ 전략 활성화/비활성화 UI 및 실시간 상태 표시 [cite: 13]                                  |

## 3. 핵심 프로세스 (Core Process Flow)

### 3.1 전략 활성화 및 시그널링

사용자: 클라이언트 UI에서 특정 전략을 활성화.

BE: 활성화된 전략 상태를 Redis에 업데이트.

Python: Redis를 읽어 현재 유저가 어떤 전략을 사용 중인지 실시간으로 파악.

Python: 시장 지표 분석 중 매수/매도 시그널 발생 시 Kafka로 이벤트 전송 및 Bybit 거래 실행.

BE: Kafka 컨슈머를 통해 시그널을 수신하고 유저에게 알림 및 DB 갱신 수행.

### 3.2 테스트 및 검증

요청: 백엔드에서 파이썬 엔진으로 특정 전략에 대한 백테스트 또는 실시간 테스트 결과 요청.

응답: 파이썬 엔진은 계산된 승률, 수익률 데이터를 반환하여 전략의 유효성 검증.

## 4. 기술적 상세 구현 계획

인증/인가: OAuth2(Google)를 통한 안전한 로그인 처리 및 사용자별 권한 관리.

지표 엔진: 앤드류스 피치포크(Andrews' Pitchfork) 기하학 로직을 통한 정밀한 추세선 분석.

실시간성 보장: Kafka를 활용한 비동기 이벤트 처리로 시그널 전송 지연 최소화.

데이터 정합성: Redis(휘발성/상태)와 RDB(영속성/이력)의 적절한 분리 사용.

5. 향후 일정 (Roadmap)
   Phase 1: 기본 아키텍처 구축 (BE-Python-Bybit 연동).

Phase 2: 지표(Pitchfork) 고도화 및 Redis 전략 동기화 구현.

Phase 3: Kafka 기반 시그널 전송 및 클라이언트 실시간 차트 연동.

Phase 4: 수익률 통계 및 유료 전략 유통 기능 추가.
